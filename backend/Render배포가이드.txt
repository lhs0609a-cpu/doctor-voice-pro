================================================================
  Doctor Voice Pro - Render 배포 가이드
================================================================

Render는 FastAPI에 최적화된 무료 호스팅 서비스입니다.

================================================================
단계 1: Render 계정 생성
================================================================

1. https://render.com 접속
2. GitHub 계정으로 가입

================================================================
단계 2: GitHub 저장소 생성
================================================================

1. GitHub에 새 저장소 생성
2. backend 폴더의 모든 파일 업로드

또는 Git 사용:
```
cd backend
git init
git add .
git commit -m "Initial commit"
git remote add origin [GitHub 저장소 URL]
git push -u origin main
```

================================================================
단계 3: Render에서 Web Service 생성
================================================================

1. Render 대시보드 → "New" → "Web Service"
2. GitHub 저장소 연결
3. 설정:
   - Name: doctorvoice-backend
   - Environment: Python 3
   - Build Command: pip install -r requirements.txt
   - Start Command: uvicorn app.main:app --host 0.0.0.0 --port $PORT

4. Environment Variables 추가:
   - ANTHROPIC_API_KEY = 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
   - ALLOWED_ORIGINS = *
   - DATABASE_URL = sqlite:////tmp/doctorvoice.db

5. "Create Web Service" 클릭

================================================================
단계 4: 배포 URL 받기
================================================================

배포가 완료되면:
- URL: https://doctorvoice-backend.onrender.com

================================================================
단계 5: 프론트엔드 환경 변수 업데이트
================================================================

frontend/.env.production 파일:
```
NEXT_PUBLIC_API_URL=https://doctorvoice-backend.onrender.com
```

================================================================
주의사항
================================================================

무료 플랜 제한:
- 15분 동안 요청이 없으면 서버가 sleep 모드로 전환
- 첫 요청 시 약 30초 걸림 (cold start)
- 월 750시간 무료

유료 플랜 ($7/월):
- Cold start 없음
- 항상 켜져 있음
- 더 빠른 성능

대안: Railway

5. 자동 배포 시작
